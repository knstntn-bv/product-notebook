# Работа с миграциями БД

При создании миграций БД **всегда** выполняй следующие шаги:

## Шаг 1: Создание миграции

- Создай файл миграции в формате `YYYYMMDDHHMMSS_description.sql` в `supabase/migrations/`
- Если изменение затрагивает и структуру, и данные - **раздели на 2 отдельные миграции**:
  * Первая: изменение структуры (ALTER TABLE, CREATE INDEX, ADD COLUMN и т.д.)
  * Вторая: миграция данных (UPDATE, INSERT и т.д.)
- Добавь комментарии в начале файла, объясняющие что и зачем изменяется
- Используй понятные имена для индексов: `idx_<table>_<column>`
- Всегда указывай DEFAULT значения для NOT NULL колонок при добавлении

## Шаг 2: Создание миграции отката

- Создай файл отката в формате `YYYYMMDDHHMMSS_description_rollback.sql` в `supabase/migrations/`
- Файл должен содержать SQL для полного отката изменений из основной миграции
- Включи удаление индексов, ограничений, колонок в **обратном порядке** их создания
- Если была миграция данных - включи откат данных в правильном порядке

## Шаг 3: Обновление final_full_schema.sql

- Обнови файл `supabase/migrations/final_full_schema.sql`
- Внеси изменения в соответствующие CREATE TABLE/ALTER TABLE секции
- Убедись, что схема отражает **актуальное состояние** после всех миграций
- Обнови комментарии в заголовке файла с указанием последних изменений
- Проверь, что все индексы, ограничения и триггеры также обновлены

## Формат миграций

- Используй формат: `YYYYMMDDHHMMSS_description.sql`
- Описание должно быть понятным и отражать суть изменения (snake_case)
- Всегда добавляй комментарии в начале файла с описанием изменений
- Для миграций отката используй суффикс `_rollback`

## Пример структуры миграции

```sql
-- Краткое описание изменений
-- Дополнительные детали, если нужно

ALTER TABLE public.table_name
  ADD COLUMN new_column type NOT NULL DEFAULT value;

CREATE INDEX idx_table_name_new_column ON public.table_name(new_column);
```
