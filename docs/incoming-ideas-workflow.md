# Workflow для обработки входящих идей

## Обзор

Этот документ описывает процесс работы со входящими идеями и предложениями в Product Notebook. Workflow обеспечивает единую точку входа для всех идей и структурированный процесс их обработки, включая как быстрые задачи, так и идеи, требующие полноценного Discovery.

## Принципы работы

1. **Единая точка входа**: Все входящие идеи размещаются в колонке **Inbox** на доске (Board)
2. **Регулярная фильтрация**: Периодический просмотр Inbox и классификация идей
3. **Два пути обработки**: 
   - Быстрый путь для готовых задач (Inbox → Backlog)
   - Discovery путь для идей, требующих исследования (Inbox → Discovery + Hypothesis)
4. **Гибкость результатов**: По итогам Discovery задача либо переходит в Backlog, либо заменяется новыми задачами

## Процесс работы

### Шаг 1: Прием входящей идеи

**Действие**: Создать Feature в колонке **Inbox** на странице Board

**Как сделать**:
1. Перейти на вкладку **Board**
2. Найти колонку **Inbox** (первая колонка слева)
3. Нажать кнопку **"Add"** в верхней части колонки Inbox
4. В открывшемся диалоге заполнить:
   - **Title** (обязательно): Краткое название идеи
   - **Description** (опционально): Подробное описание идеи, контекст, источник
   - **Column**: Оставить "Inbox" (по умолчанию)
   - **Linked Goal** (опционально): Если идея явно связана с существующей целью
   - **Linked Initiative** (опционально): Если идея явно связана с инициативой
5. Нажать **"Save Feature"**

**Рекомендации**:
- Не нужно сразу решать, требует ли идея Discovery — это будет сделано на этапе фильтрации
- Можно указать минимальную информацию (только Title), остальное можно добавить позже
- В Description можно указать источник идеи (пользователь, команда, аналитика и т.д.)

### Шаг 2: Регулярная фильтрация Inbox

**Частота**: Рекомендуется проводить ревью Inbox регулярно:
- **Ежедневно**: Быстрый просмотр, обработка очевидных случаев (5-10 минут)
- **Еженедельно**: Полноценный ревью всех накопившихся идей (30-60 минут)

**Процесс фильтрации**:

Для каждой идеи в Inbox принять решение:

#### Вариант А: Идея не требует Discovery

**Критерии**:
- Идея понятна и конкретна
- Решение очевидно
- Не требуется дополнительное исследование проблемы или решения
- Можно сразу приступить к проектированию или разработке

**Действие**: Переместить Feature в колонку **Backlog**

**Как сделать**:
1. Открыть Feature в Inbox (кликнуть на карточку)
2. При необходимости обновить описание, добавить связи с Goal/Initiative
3. В поле **"Board Column"** выбрать **"Backlog"**
4. Нажать **"Save Feature"**

**Альтернатива**: Можно перетащить карточку Feature из Inbox в Backlog (drag and drop)

**Дополнительно**:
- Если Feature связана с Goal на Roadmap, привязать её к соответствующей Goal
- Если Feature относится к Initiative, привязать для визуальной группировки

#### Вариант Б: Идея требует Discovery

**Критерии**:
- Неясна проблема, которую решает идея
- Неочевидно, что предложенное решение правильное
- Требуется валидация предположений
- Нужно исследовать пользовательские потребности или технические ограничения

**Действие**: 
1. Переместить Feature в колонку **Discovery**
2. Создать Hypothesis в портфеле гипотез

**Как сделать**:

**Часть 1: Перемещение в Discovery**
1. Открыть Feature в Inbox
2. В поле **"Board Column"** выбрать **"Discovery"**
3. Нажать **"Save Feature"**

**Часть 2: Создание Hypothesis**
1. Перейти на вкладку **Hypotheses**
2. Нажать кнопку **"Add Hypothesis"**
3. В открывшемся диалоге заполнить:
   - **Insight**: Скопировать описание идеи из Feature (или переформулировать)
   - **Status**: Выбрать **"New"** или сразу **"In Progress"** (если начинаете работу)
   - **Problem Hypothesis**: Сформулировать гипотезу о проблеме (если уже понятна)
   - **Solution Hypothesis**: Сформулировать гипотезу о решении (если уже понятна)
   - **Impact Metrics**: Указать метрики, на которые может повлиять решение (опционально)
4. Нажать **"Save Hypothesis"**

**Рекомендации**:
- Feature в Discovery служит визуальным маркером активной работы
- Hypothesis хранит структурированную информацию о валидации
- Можно создать Hypothesis сразу или после первичного анализа идеи
- В Hypothesis можно ссылаться на Feature (например, в Insight указать "См. Feature XXX-N")

### Шаг 3: Работа над Discovery

**Процесс**:
1. Feature остается в колонке **Discovery** (визуальный индикатор активной работы)
2. Hypothesis обновляется по мере проведения валидации:
   - Формулируются Problem Hypothesis и Solution Hypothesis
   - Собирается валидация (Problem Validation, Solution Validation)
   - Указываются Impact Metrics
   - Статус меняется на **"In Progress"**

**Как обновить Hypothesis**:
1. Перейти на вкладку **Hypotheses**
2. Кликнуть на строку с нужной Hypothesis
3. В открывшемся диалоге:
   - Заполнить **Problem Hypothesis** и **Problem Validation** (ссылки на исследования, данные, интервью)
   - Заполнить **Solution Hypothesis** и **Solution Validation** (прототипы, A/B тесты, эксперименты)
   - Добавить **Impact Metrics** (метрики из Strategy page)
   - Обновить **Status** по мере прогресса
4. Нажать **"Save Hypothesis"**

**Рекомендации**:
- Валидация может включать ссылки на документы, результаты опросов, аналитику
- Можно использовать статусы: New → In Progress → Accepted/Rejected
- Регулярно обновляйте Hypothesis по мере получения новых данных

### Шаг 4: Завершение Discovery

После завершения валидации принять решение о дальнейших действиях:

#### Сценарий А: Гипотеза подтвердилась, исходная Feature остается актуальной

**Действие**:
1. Обновить статус Hypothesis на **"Accepted"**
2. Обновить Feature в Discovery, добавив результаты валидации
3. Переместить Feature в **Backlog**

**Как сделать**:
1. **Hypothesis**: 
   - Открыть Hypothesis
   - Изменить **Status** на **"Accepted"**
   - Сохранить
2. **Feature**:
   - Открыть Feature в колонке Discovery
   - Обновить **Description**, добавив результаты валидации из Hypothesis
   - Изменить **Board Column** на **"Backlog"**
   - При необходимости привязать к Goal или Initiative
   - Сохранить

#### Сценарий Б: Discovery показал другую картину, нужны новые задачи

**Действие**:
1. Обновить статус Hypothesis на **"Accepted"** или **"Rejected"**
2. Исходную Feature в Discovery либо удалить, либо переместить в **Cancelled**
3. Создать новые Features на основе валидированной гипотезы

**Как сделать**:

**Вариант 1: Использовать кнопку "Create Feature" из Hypothesis**
1. Открыть Hypothesis со статусом **"Accepted"**
2. В правой панели диалога нажать **"Create Feature"**
3. В открывшемся диалоге создания Feature:
   - **Title** будет предзаполнен из Insight
   - **Description** будет предзаполнен из Solution Hypothesis
   - **Column** по умолчанию "Backlog" (можно изменить)
   - При необходимости привязать Goal или Initiative
4. Нажать **"Create Feature"**
5. Повторить для каждой новой задачи, если их несколько

**Вариант 2: Создать Features вручную**
1. Перейти на Board
2. Создать новые Features в нужных колонках (Backlog или Discovery, если нужна дополнительная проработка)
3. В Description указать связь с Hypothesis (например, "Создано на основе Hypothesis #X")

**Затем обработать исходную Feature**:
- Если Feature больше не актуальна: открыть → нажать **"Delete"** → подтвердить
- Если нужно сохранить для истории: открыть → изменить **Board Column** на **"Cancelled"**

#### Сценарий В: Гипотеза не подтвердилась

**Действие**:
1. Обновить статус Hypothesis на **"Rejected"**
2. Feature в Discovery переместить в **Cancelled** или удалить

**Как сделать**:
1. **Hypothesis**: Открыть → изменить **Status** на **"Rejected"** → сохранить
2. **Feature**: Открыть → изменить **Board Column** на **"Cancelled"** → сохранить (или удалить)

**Рекомендации**:
- Не удаляйте Hypothesis со статусом Rejected — они полезны для обучения и истории
- В Feature можно добавить краткую заметку о причинах отмены

## Дополнительные рекомендации

### Использование Goals для приоритизации

После перемещения Features в Backlog рекомендуется:
1. Просмотреть Goals на странице **Roadmap**
2. Привязать Features к соответствующим Goals
3. Это помогает видеть, какие задачи работают на конкретные цели

**Как привязать**:
- Открыть Feature
- В поле **"Linked Goal"** выбрать соответствующую Goal
- При выборе Goal автоматически устанавливается связанная Initiative
- Сохранить

### Использование Initiatives для группировки

Используйте Initiatives для визуальной группировки связанных Features:
1. На странице **Strategy** создайте или выберите Initiative
2. При создании/редактировании Feature привяжите её к Initiative
3. Features с одной Initiative будут визуально выделены цветом Initiative

### Регулярность процессов

**Ежедневно (5-10 минут)**:
- Быстрый просмотр Inbox
- Перенос очевидных задач в Backlog
- Запуск Discovery для простых случаев

**Еженедельно (30-60 минут)**:
- Полноценный ревью всех идей в Inbox
- Обновление Hypothesis в процессе Discovery
- Завершение Discovery и создание новых задач

**По завершении Discovery**:
- Обновление Features
- Создание новых задач на основе результатов
- Привязка к Goals и Initiatives

## Визуализация workflow

```
Входящая идея
    ↓
[Inbox] ← все идеи попадают сюда
    ↓
    ├─→ Не требует Discovery
    │       ↓
    │   [Backlog] ← готовая задача
    │
    └─→ Требует Discovery
            ↓
        [Discovery] + [Hypothesis: New/In Progress]
            ↓
        Валидация (обновление Hypothesis)
            ↓
    ┌───────┴───────┐
    ↓               ↓
[Accepted]      [Rejected]
    ↓               ↓
    ├─→ Обновить Feature      [Cancelled]
    │   в [Discovery]
    │       ↓
    │   [Backlog]
    │
    └─→ Создать новые Features
            ↓
        [Backlog]
```

## Часто задаваемые вопросы

**Q: Можно ли создать Hypothesis без Feature в Discovery?**  
A: Да, но рекомендуется создавать Feature в Discovery для визуального отслеживания активной работы.

**Q: Что делать, если в процессе Discovery появилось несколько разных решений?**  
A: Создайте несколько Features из одной Hypothesis или создайте дополнительные Features вручную, указав связь с Hypothesis.

**Q: Нужно ли удалять Feature из Discovery после создания новых Features?**  
A: Да, если исходная Feature больше не актуальна. Либо переместите её в Cancelled для истории.

**Q: Как отслеживать связь между Feature и Hypothesis?**  
A: В текущей версии нет прямой связи в БД. Рекомендуется:
- В Description Feature указывать ID Hypothesis (например, "Связано с Hypothesis #3")
- В Hypothesis в Insight или Solution Validation указывать ID Feature (например, "См. Feature PRO-5")

**Q: Что делать с идеями, которые не подходят ни под один из путей?**  
A: Можно оставить в Inbox для дальнейшего рассмотрения или переместить в Cancelled, если точно неактуальны.
