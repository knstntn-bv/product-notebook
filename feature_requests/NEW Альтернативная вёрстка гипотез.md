Проблема: inline редактирование гипотез даже сейчас не очень удобно. Плюс есть риск потерять правки, если не нажать Save.

Решение: реализовать редактирование гипотез при помощи диалога, аналогичного целям и фичам. В самой таблице остаются только отображение контента, изменение статуса, генерация фичи.

При отображении контента содержимое поля "Валидация проблемы" следует располагать в одной колонке с "Проблемной гипотезой", ниже неё. Так же содержимое поля "Валидация решения" следует располагать в одной колонке с "Гипотезой решения", ниже.

Вариант: вертикальный стек, с возможностью перетаскивать их вручную

Реализовано:
- Диалоговое редактирование гипотез через EntityDialog
- Объединение полей Problem Hypothesis с Problem Validation в одну колонку
- Объединение полей Solution Hypothesis с Solution Validation в одну колонку
- Статус отображается как текст (редактирование только в диалоге)
- Автоматическое распределение ширины колонок
- Удаление через диалог с подтверждением
- Все изменения описаны в docs/hypotheses-page.md
