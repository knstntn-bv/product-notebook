# Проблема

В настоящий момент инициативы существуют сами по себе, просто как обозначения направления работы. Приземление на значимые показатели появляется только в момент создания целей в рамках инициатив.

# Решение

Дать возможность указать для каждой из инициатив свою локальную North Star метрику.

# Реализация

## Данные

Для каждой инициативы появляется дополнительное поле "Целевая метрика", связывающее её с одной из существующих сущностей-метрик. Поле необязательное, значения по умолчанию нет.

## Управление

Поле может быть заполнено или изменено в любой момент, при создании или редактировании инициативы. При редактировании поля следует использовать ручной набор с автодополнением (на основании списка имеющихся метрик).

## На что влияет

Целевую метрику следует отображать в списке инициатив на странице стратегии.

Каких-то ограничений или приоритетов при создании целей в рамках инициативы в рамках этой задачи не предполагается.

---

## Реализовано

**Дата:** 2025-01-09

**Реализовано:**
- Добавлено поле `target_metric_id` в таблицу `initiatives` (nullable FK на `metrics.id`)
- Поле отображается в попап-редакторе инициативы (правая колонка, Select с опцией "None")
- Целевая метрика отображается в таблице инициатив на странице Strategy в отдельной колонке "Target Metric"
- При отсутствии метрики отображается "—"
- Реализовано совместно с ESS-64 и NEW в рамках единой задачи
