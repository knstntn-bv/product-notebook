#Проблема

В большинстве диалогов уже накопилось достаточно много опций, они не помещаются в текущую вёрстку.

# Решение

Разделить диалог редактирования на 2 части, расположенные рядом друг с другом. Части должны быть неравной ширины - соотношение примерно 70/30. Общая ширина диалога должна быть увеличена по сравнению с текущей.

1. В левой части выводятся все поля для полностью свободного ввода и свободного ввода с автодополнением (как для метрик), актуальные для редактируемой сущности

2. В правой части выводятся, сверху вниз:
2.1. Элементы выбора в виде выпадающих меню, актуальных для редактируемой сущности (такие как линки к цели, к инициативе, колонка, статус, квартал)
2.2. Кнопки, зависящие от редактируемой сущности (такие как archive, export)
2.3. Кнопки, общие для всех сущностей - delete
2.4. Отдельно, в нижнем углу диалога - тоже общие кнопки (cancel, save)

## Мобильная версия

В мобильной версии экран слишком узкий для расположения бок-о-бок, поэтому
- выпадающие меню из 2.1. следует располагать друг под другом, ниже полей ввода из пункта 1.
- пункты 2.2.-2.4. следует располагать друг под другом.
