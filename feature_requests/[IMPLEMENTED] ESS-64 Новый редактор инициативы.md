# Проблема

1. У инициативы появляется достаточно много полей, inline правка их становится уже перегруженной
2. Инициатива - стратегический элемент, он и не должен часто меняться, inline правка даже избыточна

# Решение

Реализовать отдельный попап редактора инициативы

# Реализация

В качестве образца можно использовать существующий редактор (для фич, целей, гипотез)

В левой части следует разместить название и описание

В правой части - приоритет, линк к метрике, выбор цвета, опцию архивирования.

---

## Реализовано

**Дата:** 2025-01-09

**Реализовано:**
- Создан попап-редактор инициативы на базе `EntityDialog` с двухколоночным layout
- Левая колонка: название (Input) и описание (Textarea)
- Правая колонка: приоритет (Input type="number"), целевая метрика (Select), цвет (ColorPicker на всю ширину)
- Кнопки: Save, Cancel, Archive/Unarchive, Delete
- Удалено inline-редактирование из таблицы
- Таблица стала только для просмотра, редактирование по клику на строку
- Реализовано совместно с ESS-63 и NEW в рамках единой задачи
