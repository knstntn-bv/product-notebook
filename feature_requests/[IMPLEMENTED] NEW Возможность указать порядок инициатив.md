# Проблема

Инициативы сейчас отображаются в порядке создания, что не отражает их фактического приоритета

# Решение

Реализовать возможность прямого указания приоритета инициативы

# Реализация

## Структура данных

У каждой инициативы появляется обязательное к заполнению поле приоритета. Значение - число. Значение по умолчанию - 3.

## Управление

Значение можно изменить в произвольный момент, как при создании инициативы, так и при редактировании.

## На что влияет

1. Отображение инициатив на странице стратегии - список сортируется по приоритету от меньшего к большему (в случае совпадения приоритетов - в произвольном порядке)

2. Отображение инициатив на странице роадмапа - строки таблицы сортируются по приоритету от меньшего к большему (в случае совпадения приоритетов - в произвольном порядке)

---

## Реализовано

**Дата:** 2025-01-09

**Реализовано:**
- Добавлено поле `priority` в таблицу `initiatives` (integer, NOT NULL, DEFAULT 3)
- Поле отображается в попап-редакторе инициативы (правая колонка, Input type="number" без стрелочек)
- Сортировка по приоритету реализована:
  - На странице Strategy: сначала по `priority ASC`, затем неархивные перед архивными
  - На странице Roadmap: сначала по `priority ASC`, затем неархивные перед архивными
- В ProductContext изменена сортировка загрузки инициатив с `created_at` на `priority ASC`
- Реализовано совместно с ESS-63 и ESS-64 в рамках единой задачи
