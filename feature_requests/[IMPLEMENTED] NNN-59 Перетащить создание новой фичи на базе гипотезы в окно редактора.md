# Проблема

Сейчас для создания фичи из гипотезы используется кнопка, размещённая непосредственно в столбце таблицы. У этого решения есть 2 недостатка:
- это единственный Action в таблице, неконсистентно;
- слишком накладно с точки зрения занимаемого пространства в и так громоздкой таблице.

# Решение

1. Убрать кнопку и столбец Actions из таблицы гипотез.
2. Разместить кнопку в правой панели редактора гипотез, в блоке кнопок. Поведение (в рамках этого требования) останется прежним.

Реализовано:
- Удалён столбец "Actions" из таблицы гипотез (убраны TableHead и соответствующие TableCell)
- Удалена кнопка создания фичи из таблицы гипотез
- Добавлена кнопка "Create Feature" в правую панель редактора гипотез (EntityDialog rightContent), размещена после поля выбора статуса
- При нажатии кнопки закрывается диалог редактирования гипотезы и открывается диалог создания фичи с предзаполненными данными (insight → title, solution_hypothesis → description)
- Увеличена высота поля описания в диалоге создания фичи до 15 строк
- Оптимизирован столбец Status в таблице гипотез: установлена фиксированная ширина 90px, таблица переведена на table-fixed для строгого контроля ширины столбцов
